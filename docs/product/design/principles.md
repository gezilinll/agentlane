# Design Principles

Agentlane 是 Agent Network 控制平面。设计目标是让分散在设备、runtime、agent、channel 和工作项里的运行状态变得可观察、可管理、可治理。

## Product Posture

- Agentlane 是生产工具，不是聊天产品、玩具页面或普通 SaaS 营销壳。
- 页面应优先帮助用户判断现状、定位问题、执行下一步，而不是展示概念。
- 首页和身份页可以强化品牌表达；Console 页面必须保持扫描效率、稳定布局和信息密度。
- 所有可点击入口都必须指向已经实现并可验收的能力。

## Design Order

当规则冲突时按以下优先级判断：

1. 正确表达产品对象和状态。
2. 保证当前任务可完成、可理解、可回退。
3. 保证文本、数据和交互在不同尺寸下不溢出、不遮挡。
4. 保持 Cream Arcade 视觉一致性。
5. 增加克制的游戏感和品牌记忆点。

## Simplicity

- 若无必要，勿增实体。没有明确用途的按钮、入口、图表、状态和装饰都应删除。
- 不为未来页面提前设计完整组件族；但当前已经出现的组件必须统一规范。
- 不把平台内部实现、调试证据、适配器限制直接暴露给用户。
- 不为了某个平台的字段临时污染 Agentlane 自己的数据语义。

## Brand And Product Balance

- Brand Surface 负责建立记忆点，可以大胆使用像素标题、装饰、强对比和游戏化构图。
- Product Surface 负责高频使用，像素风作为结构和强调，不牺牲可读性。
- 同一个视觉 token 系统贯穿所有页面，视觉强度由页面类型控制。

## Anti-Patterns

- 泛 SaaS 蓝白卡片堆叠。
- 玻璃拟态、渐变大字、无意义光斑、装饰性指标墙。
- 页面中出现未实现能力入口。
- Debug 字段、raw id、平台英文解释直接出现在用户界面。
- 长标题撑破布局、横向滚动、截断规则不一致。
- 多套 icon 风格混用。
- 卡片套卡片，或者页面区域全部做成浮动卡片。
